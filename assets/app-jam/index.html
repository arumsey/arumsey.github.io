Maybe a programmer is defined by the programming languages they know. Let's look at the history of the languages I've worked with over my career. I certaintly don't define myself by languages I happen to be using at any point so there must be more to being a software developer then.
Maybe an analogy will help. I particularily like comparing software development to painting a picture. This is probably due to the fact that I can't draw to save my life. First as a UI developer it is my duty to paint a picture on your screen that makes and applcation enjoyable and easy to use. Second, just like a painting, software consists of many layers and any one of those layers on its own may not do a lot but when you combine many of these layers together the result is something that is much more useful.
OK, enough of the analogies. How about an acutal definition.

Products

So what exactly do I do here at Adobe? As a developer I contribute to one of our marketing cloud solutions called Expereince Manager as well as our PhoneGap technology. More specifically the team here in Ottawa is building a mobile app development platform that can be used by large organizations. The plaform we are building integrates with PhoneGap as well as other app types and makes it easier to manage the entire mobile app development lifecycle.

Process

In order to create that software "painting" we follow an agile development process which looks something like this. During this process requirements can be added at any time and releases can be made. Interestingly the implementation of a requirement or the actual development time is only a small portion of the entire process. The great thing about such a process is that it can be followed by anyone whether they are part of a 100 person team or if they are hacking with a friend on the weekend.

Tools

In order to succesfiully follow such a process we will need some tools. Here at Adobe we rely on wikis for documenting requirements. Requirements are then added to an issue tracker which allows developers to be assigned to particular features or tasks from which work can be logged. Finally a source control system like Git ensures multiple developers can easily contribute to the same project while maintaining the software's entire history. All of these tools in some form are free to use which allows developers at any level to begin following an agile process.
